Start the Scan

Click the ‚ö° Scan for Deleted Files button at the bottom of the sidebar.

IronRod uses two scanning strategies automatically:

  • Forensic Mode ‚Äî Parses the filesystem's allocation bitmap to scan only unallocated (deleted) space. Faster and more accurate. Works on exFAT, FAT32, NTFS, ext2/3/4, and more.
  • Brute-Force Mode ‚Äî Scans every sector of the disk byte-by-byte. Used when the filesystem is missing, corrupted, or unsupported. Works on any device.

During the scan, you'll see:

  • A progress bar showing percentage complete
  • Scan speed in MB/s
  • ETA (estimated time remaining)
  • File count updating in real-time as files are found
  • Scan mode indicator (Forensic or Brute-Force)
üîí
Your source drive is never modified. IronRod opens the device in read-only mode. It's safe to scan even if you're not sure it's the right drive.

You can ‚èπ Cancel the scan at any time. Files found so far will still be available.

Using CLI Mode

IronRod also supports a command-line interface for advanced users, scripting, and headless servers.

Terminal $ sudo ironrod --cli -d /dev/disk4

CLI Options:

Options --cli             Run in command-line mode (no GUI)
-d, --device       Path to disk device or image file
-o, --output       Output directory (default: ~/Desktop/RecoveredFiles)
-c, --categories   Comma-separated list: Image,Video,Audio,Document
--no-elevate       Skip auto-elevation (already running as root)

Examples:

macOS / Linux # Scan a USB drive, recover only images and videos
sudo ironrod --cli -d /dev/sdb -c Image,Video

# Scan a disk image file
sudo ironrod --cli -d ~/forensic-copy.dd -o ~/recovered

# Scan with all defaults
sudo ironrod --cli -d /dev/disk2
Windows (Admin CMD) IronRod.exe --cli -d \\.\PhysicalDrive1

REM Scan a specific partition
IronRod.exe --cli -d E:

Pro Tips

‚è±Ô∏è
Act fast! The sooner you scan after deletion, the better your chances. Every second the drive is in use, deleted data may be overwritten.
üíΩ
SSDs with TRIM: Modern SSDs erase deleted data within seconds via TRIM. IronRod detects this and warns you. If TRIM has run, the data is physically gone ‚Äî no software can recover it.
üìã
Disk images: For critical recovery, create a full disk image first using dd and scan the image instead. This preserves the original drive state.
üéØ
Best results: Use IronRod on an HDD (not SSD), scan immediately after deletion, and save to a completely different physical drive. This gives you the highest recovery rate.
üìä
Large drives: For drives larger than 500 GB, uncheck file categories you don't need. This reduces scan time significantly. A 1 TB HDD can take 20-60 minutes for a full scan.

Frequently Asked Questions

Is IronRod really free? ‚ñº
Yes, 100% free. No trial limitations, no watermarks, no hidden costs. You can recover unlimited files from unlimited drives.
Will IronRod damage my drive? ‚ñº
No. IronRod opens your drive in strict read-only mode. It never writes a single byte to your source drive. It's completely safe to use.
Why does it need admin/root access? ‚ñº
Operating systems protect raw disk access behind admin privileges for security. IronRod needs this to read the raw sectors where deleted file data still exists. Without admin access, it can only see the normal filesystem which has already "forgotten" the deleted files.
Can I recover files from an SSD? ‚ñº
It depends. Modern SSDs use TRIM to erase deleted data within seconds. If TRIM has already run, the data is physically zeroed out ‚Äî no software can recover it. IronRod will detect this and warn you. HDDs, USB drives, and SD cards don't have TRIM, so recovery is much more likely.
What filesystems does it support? ‚ñº
Forensic mode works on exFAT, FAT12/16/32, NTFS, ext2/3/4, HFS+, APFS, Btrfs, XFS, F2FS, UDF, and ISO 9660. Brute-force mode works on ANY device ‚Äî even ones with no filesystem at all.
Why are some recovered files corrupted? ‚ñº
When a file is deleted, its disk space is marked as available. If new data was partially written over that space, the recovered file will be corrupted. IronRod detects this and marks files as "Damaged". It also attempts automatic repair for common formats (JPEG, MP4, PNG, etc.).
How long does a scan take? ‚ñº
It depends on the drive size and type. Forensic mode scans only unallocated space, which is much faster. A 256 GB USB drive typically takes 5-15 minutes. A 2 TB HDD can take 30-90 minutes. IronRod shows real-time speed and ETA during the scan.
